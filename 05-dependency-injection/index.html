Dependency Injection Demo

Dependency injection is an example of what we may call inversion of control. It's a design pattern that allows us to decouple the creation of objects from their dependencies. This is useful for modifiability and maintenance.

In this demo, we'll use dependency injection to create a web component that uses a different storage implementations. We'll also use a factory to create the storage implementation.

Memory Storage

Data stored in memory - resets on page reload, not secure, quite performant

LocalStorage

Data persisted in browser's localStorage - survives page reload, not secure, quite performant
